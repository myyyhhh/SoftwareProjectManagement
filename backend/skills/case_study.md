# 案例研究 (Case Study) 撰写规范

## 1. 结构要求（STAR / PSR 法则）
- **背景/挑战 (Problem/Situation)**: 描述客户（或典型用户）在遇到我们产品之前面临的核心痛点、业务瓶颈或行业挑战。
- **解决方案 (Solution/Task & Action)**: 详细说明我们的产品或服务是如何介入并解决客户问题的。请针对性地解释所用的核心功能及其独特优势。
- **成效/结果 (Result)**: 必须要提供具体的、可量化的数据支持（例如：效率提升xx%、成本降低xx%、销售额翻倍、活跃用户暴增）。
- **客户引言 (Quote)**: 用一句强有力且真诚的客户语录，高度赞扬产品或服务对他们带来的实际帮助（如果没有原始语录，可基于案例合理拟写预期的话术框架，标明 [客户引言占位符]）。
- **结论与行动号召 (CTA)**: 呼吁面临类似困境的潜在客户与我们取得联系并体验产品。

## 2. 品牌一致性与基调
- **语气**: 理性、客观、权威、值得信赖、具有商业洞察力。
- **数据导向**: 结果尽可能用客观数据表现，避免含糊其辞及虚假宣传。
- **客户视角为主**: 案例应以客户的成功经历为主线，产品扮演“赋能者”和“催化剂”的陪伴角色，避免全篇自夸和过度自我营销。
